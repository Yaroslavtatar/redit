# CREDDIT — мини‑Reddit на Flask

CREDDIT — это учебный мини‑клон Reddit, созданный на **Flask** с использованием
только **HTML**, **CSS**, **JavaScript** и **SQLite**. Проект ориентирован на
новичков: код снабжён развёрнутыми комментариями, а данное руководство подробно
объясняет архитектуру и ключевые технологии.

---

## Возможности

- **Регистрация и вход** пользователя (аутентификация на Flask через сессии).
- **Создание постов** авторизованными пользователями.
- **Просмотр ленты постов** на главной странице.
- **Детальная страница поста** с полным текстом.
- **Комментарии** к постам (создание и просмотр).
- **Голосование (лайк/дизлайк)** за посты с пересчётом рейтинга **без
  перезагрузки страницы** (через fetch в JavaScript).
- **Хранение данных в SQLite**, чтобы всё сохранялось между перезапусками.
- **Синяя цветовая гамма** и единый стиль интерфейса.

---

## Технологии

- **Python 3 + Flask** — веб‑фреймворк для обработки HTTP‑запросов,
  маршрутизации и работы с шаблонами.
- **HTML5 + Jinja2** — разметка страниц и шаблоны с подстановкой данных из
  Python‑кода.
- **CSS3** — оформление в синей палитре, адаптивная вёрстка.
- **Vanilla JavaScript** — небольшая логика на фронтенде (голосование).
- **SQLite** — лёгкая встроенная СУБД, данные хранятся в обычном файле.

## Установка и запуск

### 1. Установка Python и зависимостей

1. Убедитесь, что установлен Python 3 (проверьте командой `python --version`
   или `py --version` в Windows PowerShell).
2. (Рекомендуется) создайте виртуальное окружение:

   ```bash
   python -m venv .venv
   .venv\Scripts\activate  # Windows (PowerShell)
   ```

3. Установите зависимости из `requirements.txt`:

   ```bash
   pip install -r requirements.txt
   ```

### 2. Инициализация базы данных

CREDDIT использует SQLite‑базу данных, путь к которой задаётся в `config.py`
(значение `DATABASE_PATH`). Чтобы создать файл БД и таблицы:

```bash
flask --app app.py init-db
```

Эта команда вызывает функцию `init_db()` из `db.py`, которая создаёт таблицы:

- `users` — пользователи;
- `posts` — посты;
- `comments` — комментарии;
- `votes` — голоса (лайки/дизлайки).

### 3. Запуск сервера разработки

```bash
python app.py
```

По умолчанию Flask поднимет сервер по адресу `http://127.0.0.1:5000/`.
Откройте этот адрес в браузере — вы попадёте на главную страницу CREDDIT.

## Архитектура бекенда

### Flask-приложение (`app.py`)

Главная функция — `create_app()`. Она:

- создаёт объект `Flask`;
- подгружает конфиг (`SECRET_KEY`, `DATABASE_PATH` из `config.py`);
- регистрирует функции работы с БД (через `app.teardown_appcontext(close_db)`);
- добавляет CLI-команду `init-db` для инициализации БД;
- описывает маршруты (routes) приложения.

Ключевые маршруты:

- `GET /` — главная страница, список постов.
- `GET /register`, `POST /register` — регистрация.
- `GET /login`, `POST /login` — вход.
- `GET /logout` — выход.
- `GET /post/create`, `POST /post/create` — создание поста.
- `GET /post/<id>` — просмотр поста.
- `POST /post/<id>/comment` — добавление комментария.
- `POST /post/<id>/vote` — голосование за пост.

### Работа с SQLite (`db.py`)

Основные элементы:

- `get_db()` — создаёт (при первом обращении) и возвращает подключение к БД,
  привязанное к текущему запросу Flask через объект `g`.
- `close_db()` — закрывает подключение в конце запроса (Flask вызывает
  автоматически).
- `execute(query, params, commit=False)` — удобная обёртка над `db.execute`:
  используется для всех SQL‑запросов.
- `init_db()` — выполняет SQL‑скрипт с `CREATE TABLE IF NOT EXISTS` для всех
  таблиц.

**Почему SQLite?**

- Никакого отдельного сервера базы данных поднимать не нужно.
- Данные хранятся в обычном файле (`creddit.db`), который лежит рядом с кодом.
- Для учебного мини‑проекта этого более чем достаточно.

## Модель данных

### Таблица `users`

- `id` — первичный ключ.
- `username` — уникальное имя пользователя.
- `password_hash` — **хэш пароля** (а не сам пароль).
- `created_at` — дата создания записи.

### Таблица `posts`

- `id` — первичный ключ.
- `user_id` — автор поста (внешний ключ на `users.id`).
- `title` — заголовок поста.
- `content` — текст.
- `created_at` — дата создания.

### Таблица `comments`

- `id` — первичный ключ.
- `post_id` — пост, к которому относится комментарий.
- `user_id` — автор комментария.
- `content` — текст.
- `created_at` — дата создания.

### Таблица `votes`

- `id` — первичный ключ.
- `post_id` — пост, за который голосуют.
- `user_id` — кто голосует.
- `value` — значение голоса: `1` (лайк) или `-1` (дизлайк).
- `created_at` — дата создания голоса.
- `UNIQUE (post_id, user_id)` — один пользователь может иметь только один
  активный голос за конкретный пост.

Рейтинг поста считается как сумма `value` по всем строкам для данного `post_id`.

## Поток запроса (упрощённо)

Когда пользователь заходит на любую страницу CREDDIT, происходит примерно
следующее:

1. Браузер отправляет HTTP‑запрос (GET или POST) на сервер.
2. Flask ищет подходящий маршрут (например, `/`, `/login`, `/post/1`).
3. Внутри обработчика маршрута:
   - при необходимости вызывается `execute()` для работы с БД;
   - собираются данные (список постов, один пост, комментарии и т.п.).
4. Функция `render_template()` подставляет эти данные в HTML‑шаблон.
5. Сформированный HTML возвращается браузеру, и пользователь видит страницу.

## Аутентификация и сессии

Для регистрации и входа используется стандартный подход Flask:

- при регистрации пароль **никогда не хранится в открытом виде**;
  используется `generate_password_hash()` из `werkzeug.security`;
- при входе пароль, введённый пользователем, проверяется через
  `check_password_hash()`;
- после успешного входа в объект `session` записываются `user_id` и
  `username` — так сервер \"помнит\", кто залогинен.

Пример: при создании поста маршрут `/post/create` проверяет, есть ли
`session["user_id"]`. Если нет — пользователя перенаправляют на `/login`.

## Голосование и JavaScript

Голосование реализовано без перезагрузки страницы:

- На странице поста (`post_detail.html`) есть блок с кнопками:
  - \"👍 Лайк\" (`data-value="1"`)
  - \"👎 Дизлайк\" (`data-value="-1"`)
- В `static/js/main.js` функция `setupVoting()`:
  - находит контейнер с кнопками;
  - вешает обработчик клика;
  - по клику отправляет `fetch('/post/<id>/vote')` с JSON‑телом
    `{ "value": 1 }` или `{ "value": -1 }`;
  - по успешному ответу (`{ ok: true, score: <новый рейтинг> }`) обновляет
    число в элементе `#post-score`.

На сервере маршрут `POST /post/<id>/vote`:

- проверяет, что пользователь залогинен;
- принимает и валидирует `value`;
- создаёт или обновляет запись в таблице `votes`;
- пересчитывает рейтинг через `SELECT SUM(value) FROM votes WHERE post_id = ?`;
- возвращает результат в формате JSON.

## Синяя цветовая схема и UI

Основной стиль описан в `static/css/main.css`:

- В корне файла определены CSS‑переменные:
  - `--blue-main`, `--blue-main-dark`, `--blue-main-light` — разные оттенки
    синего для фона, кнопок и hover‑состояний.
  - Переменные для фона (`--bg-body`, `--bg-card`), текста и границ.
- Шапка (`.site-header`) реализована с градиентом от тёмно‑синего к
  основному синему.
- Лента постов и карточки постов используют белый фон и мягкую тень, чтобы
  контент читался комфортно.
- Формы аутентификации и создания поста оформлены как аккуратные карточки
  с понятными подписями и подсказками.

## Как пользоваться приложением

1. Откройте `http://127.0.0.1:5000/` — увидите пустую ленту (если ещё нет постов).
2. Перейдите к регистрации:
   - нажмите \"Регистрация\" в правом верхнем углу;
   - введите имя и пароль;
   - после успешной регистрации вас перенаправит на страницу входа.
3. Войдите в систему через \"Вход\".
4. Создайте пост:
   - в шапке появится ссылка \"Создать пост\";
   - заполните заголовок и текст;
   - после сохранения пост появится в ленте.
5. Откройте пост, чтобы:
   - прочитать его полностью;
   - оставить комментарий (если вы залогинены);
   - поставить лайк или дизлайк.

## Куда смотреть новичку в коде

Если вы только начинаете разбираться с Flask, рекомендуется идти в таком
порядке:

1. `app.py` — познакомиться с маршрутизацией и функцией `create_app()`.
2. `templates/base.html` — понять, как устроен базовый HTML‑каркас, как
   подключаются стили и скрипты.
3. `templates/index.html` — увидеть, как выводится список постов с помощью
   Jinja2 (`{% for post in posts %}`).
4. `db.py` — посмотреть, как создаётся и инициализируется SQLite‑база.
5. `templates/post_detail.html` + `static/js/main.js` — связка шаблона и
   JavaScript для голосования и комментариев.

Все эти файлы снабжены подробными комментариями, которые объясняют не только
\"что\" делает код, но и **почему** он устроен именно так.
